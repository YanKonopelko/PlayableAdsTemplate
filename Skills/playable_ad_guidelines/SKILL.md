# Разработка Playable Ads (Cocos Creator 3.8.7)

При написании кода для Playable Ads необходимо учитывать следующие ограничения и практики:

1. **Единый файл / Инлайн**:
   Конечный билд часто должен быть собран в один HTML файл. Избегай подгрузки ассетов из сети в процессе игры, если это не разрешено конкретной рекламной сетью (Ad Network).
2. **Логика перехода в стор (`ToStore`)**:
   В проекте используется метод (например, в `GameplayScene.ts`), который открывает страницу игры в App Store / Google Play. Все взаимодействия в конце игры (win/lose) или при тапе по фону/кнопкам CTA (Call to Action) должны вести к вызову этого метода. Учитывай таймер бездействия: если пользователь не проявляет активности заданное время, вызывай метод перехода в стор.
3. **Оптимизация размеров**:
   Используй минимально возможные текстуры, сжимай аудио. Учитывай, что общий размер билда (часто до 2-5 МБ) критичен. Скрипты должны быть лаконичными.
4. **Управление аудио**:
   Звуки должны корректно ставиться на паузу, когда вкладка скрывается, и возобновляться при открытии. Звук не должен воспроизводиться до первого тапа пользователя из-за политик браузеров (Audio Context Play Policy).
5. **Аналитика / События (MRAID)**:
   При необходимости интегрируй вызовы MRAID для отслеживания событий взаимодействия (custom events).
